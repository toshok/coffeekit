CoffeeKit 0.2 Release Notes

Jan 21st, 2012

Changes since the last release:

  1. The biggest change is the move away from JavaScriptCore to SpiderMonkey as our JavaScript engine. This switch has a number of important effects, including:
    • SpiderMonkey has a faster interpreter than JavaScriptCore, so nearly everything that CoffeeKit does is faster - this should be immediately obvious in the opengl demos in the sample project.
    • SpiderMonkey’s error reporting is much more mature than JavaScriptCore, and helped to find a number of errors in the bindings and sample code.
    • SpiderMonkey’s license permits applications to be released to the app store. JavaScriptCore’s license would have required application developers to be able to release the CoffeeKit driver code, which would have been impossible given that they don’t have access to it. Given the bugs remaining and features missing I don’t recommend trying to deploy to the appstore yet, but at this point it’s basically a technical issue, not a licensing one, which is always a comfort.
  2. The bindings for CoffeeKit have been rewritten using a more DSL-like approach, with support for specifying the actual types of parameters and return values. (There’s an upcoming blog post about this.)
